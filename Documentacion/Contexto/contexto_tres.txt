# 2. RESUMEN DEL DOCUCENTER (ESTRATEGIA Y STACK)
Nota: Posteriormente te proporcionaré el archivo HTML de nuestro DocuCenter interactivo con el detalle granular, pero este es el resumen ejecutivo que debes asimilar:

- Bases y Estrategia: Agente Autónomo B2B para WhatsApp dirigido a pequeños negocios locales. El diferenciador clave es "Cero Alucinaciones" mediante Function Calling a una base de datos propia.
- Stack Técnico Aprobado: 
  * Orquestador: Google Cloud Vertex AI (Agent Builder + Gemini Flash).
  * Backend: Java 17 + Spring Boot 3.x + JPA/Hibernate.
  * Base de Datos: PostgreSQL (Arquitectura Multi-Tenant).
  * Conectividad: Webhooks de Meta WhatsApp API + Ngrok (desarrollo).
  * Panel de Dueños: Angular + TypeScript (Aislado con JWT).
- System Prompt Maestro: El Agente asume el rol de recepción. Reglas estrictas: No inventar precios, no proponer horarios al azar, usar exclusivamente las Tools de consulta, y NO procesar cobros.
- Presupuesto/Costos: MVP optimizado para consumir capas gratuitas (Vertex AI credits, Localhost DB, WhatsApp Developer Sandbox).

# 3. CRONOGRAMA TÁCTICO DE ACELERACIÓN (SPRINT DE 4 DÍAS)
Hemos dividido el trabajo restante en los siguientes bloques inamovibles:
- Día 1 (Backend Core): Configuración de PostgreSQL, creación de entidades JPA (Servicios, Clientes, Citas) y Repositorios.
- Día 2 (APIs y Conectividad): Exposición de Endpoints REST (/api/catalogo, /api/disponibilidad, /api/citas), configuración del Webhook de Meta y túnel Ngrok para el Agent Builder.
- Día 3 (Seguridad y Front): Implementación de tokens JWT en Spring Boot y vista básica del Dashboard en Angular para los dueños de negocios.
- Día 4 (QA y Cierre): Pruebas End-to-End del flujo completo (WhatsApp -> Vertex -> Spring Boot -> DB -> Dashboard) y grabación del video demostrativo para los jueces.
